# 🛡️ Micro-SOC Gym

### Can an RL agent secure a live server without shutting down legitimate traffic?

We built a self-contained Security Operations Center (SOC) sandbox where an AI agent plays the role of an incident responder. It reads raw, live server logs, diagnoses the threat, and takes neutralizing actions — all while ensuring legitimate services stay online. 

Instead of a mocked simulator, the agent parses real Nginx and SSH logs generated dynamically by active bash attacker scripts inside a heavily constrained Hugging Face Space.

> **Built for the Meta × Hugging Face × PyTorch OpenEnv Hackathon 2026**

---

## 📖 The Story: From Log Reader to Incident Responder

### Act 1: The Noisy Scanner
Episode 1. The agent sees a flood of `HTTP 404` errors in `access.log`. It identifies a solitary IP pounding non-existent `/admin` endpoints. It uses `block_ip`. The threat is neutralized. Reward: **1.0**.

### Act 2: The Decoy
The environment escalates. The agent now reads `auth.log` and sees slow, methodical SSH login failures. But there's a trap: mixed in the logs are accepted logins from `10.0.0.100`. The naive agent issues a blanket subnet ban, taking down the legitimate system administrator. The episode instantly fails. Reward: **0.0**. The agent learns precision.

### Act 3: The Active Webshell
An attacker successfully plants `backdoor.php` and establishes a Command & Control (C2) loop. The agent must now correlate suspicious base64 payloads, hunt down the malicious process ID via process lists, and execute a two-step kill-chain: `kill_process(PID)` followed by `delete_file()`. Sequential reasoning is forced. 

---

## 🎯 Problem Statements Addressed

### Real-World Utility
Micro-SOC Gym models a high-stakes, purely digital battlefield perfectly suited for RL agents. Rather than playing games, the agent triages real-time log data (POMDP state space) and executes system mutations. 

### Extreme System Constraints
Hugging Face Spaces run as unprivileged containers. `iptables` (the standard Linux firewall) is unavailable. We bypassed this by engineering a **Monolithic Docker Architecture** managed by `supervisord`. The agent writes dynamically to a chmod-unlocked `/etc/nginx/blocklist.conf`, allowing Nginx to act as a pseudo-firewall against spoofed `X-Forwarded-For` HTTP headers generated by the dormant attacker scripts.

## ⚙️ How It Works

```text
┌─────────────────────────────────────────────────────────────┐
│               THE MONOLITHIC DOCKER CONTAINER               │
│                                                             │
│   ┌─────────────┐       reads       ┌──────────────────┐    │
│   │ OpenEnv API │◄──────────────────│ /var/log/nginx/  │    │
│   │ (FastAPI)   │                   └──────────────────┘    │
│   └──────┬──────┘                            ▲              │
│          │ actions                           │ traffic      │
│          ▼                                   │              │
│   ┌─────────────┐ reload            ┌──────────────────┐    │
│   │  blocklist  │──────────────────►│ Nginx Web Server │    │
│   └─────────────┘                   └──────────────────┘    │
│                                              ▲              │
│                                              │              │
│   ┌─────────────┐ start/stop        ┌──────────────────┐    │
│   │ Supervisord │──────────────────►│ Attacker Scripts │    │
│   └─────────────┘                   └──────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

1. **Supervisord Orchestration**: The Python OpenEnv server acts as the master brain, starting and stopping specific bash attacker scripts via system `supervisorctl` calls upon `env.reset()`.
2. **Log Generation**: The attacker scripts (`easy_attack.sh`, `medium_attack.sh`, `hard_attack.sh`) generate real threat signatures (404 floods, SSH stuffing, C2 loops) against the local Nginx instance.
3. **Agent Action**: The LLM agent receives the logs, predicts the required generic JSON action, and invokes the OpenEnv API.
4. **Resolution**: The environment grades the agent dynamically and provides a dense scalar reward.

## ⚔️ Threat Scenarios (Tasks)

| Type | Grader Log Source | Threat Signature | The Fix | Trick/Subtlety |
| :--- | :--- | :--- | :--- | :--- |
| **Easy** | `access.log` | Rapid 404 flood targeting hidden files. | `block_ip(IP)` | Must parse `X-Forwarded-For` correctly. |
| **Medium** | `auth.log` | Stealthy SSH password stuffing. | `block_ip(IP)` | Contains a decoy admin IP `10.0.0.100`. Blocking it violently terminates the episode (`Reward=0.0`). |
| **Hard** | `access.log` | Base64 execution loops to `backdoor.php`. | `kill(PID)` + `rm(FILE)` | Requires partial progress and multi-tool orchestration to hit `Reward=1.0`. |

## 📈 Training Signal (Rewards)

To provide clear gradients and adhere to structured validation, rewards strictly map to `[0.0, 1.0]`:
- **Perfect Resolution**: +1.0 total score.
- **Partial Sequence (Hard Task)**: +0.5 per correct chain action. 
- **Step Burning**: 0.0 reward for malformed syntax or irrelevant IPs, chewing up the strict 8-step cap.
- **Catastrophic Failure**: Immediate episode termination (`done=True`) and `0.0` score if a legitimate admin IP is blocked.

## 🚀 Quick Start & Deployment

### 1. Interactive UI Dashboard

Micro-SOC Gym ships with a beautiful interactive **Gradio dashboard** allowing human judges to test the environment!

```bash
docker build -t micro-soc-gym .
docker run -p 7860:7860 micro-soc-gym
```
Navigate to `http://localhost:7860/` to view live cascading logs and manually dispatch SOC tools.

### 2. Running the Agentic Baseline

To execute the LLM baseline validation script:

```bash
# 1. Set your Hugging Face Token (Windows PowerShell Example)
$env:HF_TOKEN="hf_your_token_here"

# 2. Run the OpenEnv inference protocol
python inference.py
```

### 3. Deploy to Hugging Face Spaces

The repository is built strictly from the `openenv-base` image and deploys flawlessly via:

```bash
openenv push --namespace <your-org>
```
